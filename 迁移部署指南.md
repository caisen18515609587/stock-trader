# StockTrader 迁移部署指南

> 本文档描述如何将整个 StockTrader 项目完整迁移到另一台机器并正常运行。

---

## 一、项目数据存储位置总览

项目所有数据均存储在项目目录内，**不依赖外部数据库或云存储**，可整体打包迁移。

```
MyProject/
├── data/
│   ├── stock_trader.db          # ⭐ 核心数据库（所有交易记录、持仓、用户数据）
│   ├── stock_trader.db-shm      # SQLite 共享内存文件（运行时产生）
│   ├── stock_trader.db-wal      # SQLite WAL 日志（包含最新未合并数据）
│   ├── stock_basic_cache.json   # 股票基础信息缓存（可重新自动生成）
│   ├── auto_trader_main.json    # 自动交易策略配置
│   ├── users.json               # 多账户用户配置
│   └── users/                   # 各账户交易报告（按账户ID分目录）
├── logs/                        # 运行日志（可不迁移）
├── reports/                     # 导出报表（可不迁移）
├── target/
│   └── stock-trader-1.0.0-SNAPSHOT-jar-with-dependencies.jar  # 编译好的程序
├── data-service/
│   ├── app.py                   # Python 数据服务
│   ├── qmt_bridge.py            # QMT 实盘交易桥接服务
│   ├── requirements.txt         # Python 依赖清单
│   └── start-data-service.sh    # 数据服务启动脚本
├── src/                         # Java 源代码
├── pom.xml                      # Maven 构建配置
├── start-trader.sh              # 自动交易启动脚本（⚠️ 含硬编码路径，需修改）
└── src/main/resources/
    └── application.properties   # 系统配置（策略参数、端口、推送等）
```

## 二、新机器环境要求

| 依赖 | 版本要求 | 说明 |
|---|---|---|
| **Java JDK** | 8（推荐 Amazon Corretto 8） | 运行 JAR 程序 |
| **Python** | 3.9 及以上 | 运行数据服务 |
| **pip 包** | 见下方清单 | 数据服务依赖 |
| **Maven** | 3.6+（可选） | 仅需重新编译源码时使用 |

Python 依赖清单（`data-service/requirements.txt`）：
```
flask>=3.0.0
tushare>=1.4.0
pandas>=2.0.0
requests>=2.28.0
```

## 三、迁移步骤

### 第1步：在旧机器上打包（重要：先停程序）

> ⚠️ **必须先停止程序再压缩**，否则 `stock_trader.db-wal` 文件可能不完整，导致数据丢失。

```bash
# 1. 停止所有相关进程
pkill -f stock-trader
pkill -f "python.*app.py"

# 等待 2~3 秒确保进程退出
sleep 3

# 2. 压缩打包（排除大日志文件以减小体积）
cd /Users/caisen/IdeaProjects
zip -r MyProject-backup.zip MyProject/ \
    --exclude "MyProject/logs/*.log" \
    --exclude "MyProject/data-service/*.log" \
    --exclude "MyProject/target/classes/*"
```

> 如需保留完整日志，去掉 `--exclude` 那两行即可。

---

### 第2步：在新机器上安装环境

```bash
# 安装 Java 8（macOS，以 Amazon Corretto 为例）
# 下载地址：https://aws.amazon.com/cn/corretto/
# 安装后验证：
java -version
# 期望输出：openjdk version "1.8.0_xxx"

# 安装 Python 依赖
cd /path/to/MyProject/data-service
pip3 install -r requirements.txt
```

---

### 第3步：解压并修改启动脚本路径

解压后，需要修改 **2个文件** 中的硬编码路径：

#### 修改 `start-trader.sh`

```bash
# 将以下两行改为新机器的实际路径
export JAVA_HOME=/Library/Java/JavaVirtualMachines/amazon-corretto-8.jdk/Contents/Home
cd /Users/caisen/IdeaProjects/MyProject
```

修改示例（假设新机器路径为 `/home/user/MyProject`）：
```bash
export JAVA_HOME=/usr/lib/jvm/java-8-amazon-corretto   # 改为新机器 Java 路径
cd /home/user/MyProject                                  # 改为新机器项目路径
```

#### 修改 `data-service/start-data-service.sh`

```bash
# 将以下行改为新机器的实际路径
cd /Users/caisen/IdeaProjects/MyProject/data-service
```

---

### 第4步：配置环境变量（推送和Token）

项目的敏感配置（微信推送、iFind Token等）通过**环境变量**注入，不存储在项目文件中。
需要在新机器的 `~/.zshrc`（macOS/Linux）或系统环境变量中重新配置：

```bash
# 编辑 ~/.zshrc 或 ~/.bashrc，添加以下内容（填入真实值）
export PUSH_MP_APPID=wx你的公众号AppID
export PUSH_MP_APPSECRET=你的AppSecret
export PUSH_MP_TEMPLATE_TRADE=你的交易模板ID
export PUSH_MP_TEMPLATE_DAILY=你的日报模板ID
export PUSH_MP_OPENID=你的OpenID
export PUSH_WECHAT_SENDKEY=你的Server酱SendKey
export DATASOURCE_THS_TOKEN=你的iFind Token

# 生效
source ~/.zshrc
```

> 如果暂时不需要推送功能，可跳过此步，系统会以静默模式运行。

---

### 第5步：启动服务

**启动顺序：先启动数据服务，再启动交易程序。**

```bash
# 1. 启动数据服务（Python，提供行情API）
cd /path/to/MyProject/data-service
bash start-data-service.sh

# 验证数据服务正常（返回 {"status":"ok"} 即成功）
curl http://localhost:8099/health

# 2. 启动自动交易程序（Java）
cd /path/to/MyProject
bash start-trader.sh

# 3. 启动 Web 管理平台（可选，用于查看持仓/报表）
nohup java -jar target/stock-trader-1.0.0-SNAPSHOT-jar-with-dependencies.jar platform >> logs/platform.log 2>&1 &
# 访问地址：http://localhost:8090
```

---

### 第6步：验证迁移是否成功

```bash
# 检查进程是否正常运行
ps aux | grep -E "stock-trader|app.py" | grep -v grep

# 检查数据库是否可以正常读取（应显示账户列表）
sqlite3 /path/to/MyProject/data/stock_trader.db \
    "SELECT user_id, name, total_assets FROM t_portfolio LIMIT 5;"

# 查看最新日志确认无报错
tail -50 logs/auto-trade.log
tail -20 logs/data-service.log
```

## 四、注意事项

| 事项 | 说明 |
|---|---|
| **停程序再压缩** | WAL 文件未合并时直接复制可能导致数据不一致 |
| **3个DB文件都要带走** | `stock_trader.db` + `.db-shm` + `.db-wal` 缺一不可 |
| **target/JAR 可不重新编译** | 直接带走编译好的 JAR 即可，无需在新机器安装 Maven |
| **Java 版本必须是 8** | 启动脚本和JAR均针对 JDK 8 编译，高版本可能有兼容问题 |
| **端口占用检查** | 数据服务用 `8099`，交易程序用 `8090`/`8888`，确保新机器端口未被占用 |
| **macOS → Linux 迁移** | `start-trader.sh` 中 `JAVA_HOME` 路径格式不同，需按 Linux 规范修改 |

## 五、不需要迁移的内容

- `logs/` 目录（日志文件，新机器会自动重建）
- `reports/` 目录（导出报表，可选带走）
- `data/cache/` 目录（K线缓存，启动后会自动重新拉取）
- `target/classes/` 目录（编译中间产物）
